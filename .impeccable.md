## Design Context

### Users
Knowledge workers who curate insights from X/Twitter into Obsidian and Logseq knowledge graphs. They're reading threads, finding signal, and building a personal knowledge base. They use the tool between reading sessions — paste a URL, get structured markdown, move on.

### Brand Personality
Cold, precise, competent. "A data pipeline with a face." No personality beyond doing the job well.

### Aesthetic Direction
Stark monochrome, futuristic minimalism inspired by X.ai's grid aesthetic. Dark mode only. Graph grid background (32px cells) that content floats above. Borders-only depth — no shadows, no gradients, sharp corners. System sans-serif for UI, monospace for the markdown output preview. Not more than 3 colors. The graph grid is the product's visual identity.

Anti-references: anything warm, playful, rounded, or "friendly." No glassmorphism, no gradients, no bounce animations.

References: X.ai grid pages, terminal UIs, technical mono trend (2026).

### Design Principles
1. **Infrastructure aesthetic** — this is a tool, not a destination. It should feel like peering into an extraction system.
2. **Signal over decoration** — every element earns its place. No ornament, no personality beyond competence.
3. **Grid as identity** — the subtle graph grid connects the tool to the knowledge graph destination.
4. **Monochrome discipline** — void (#0a0a0a), ink (#e5e5e5), and a faint cool grid tone. No accent color.
5. **Content masks grid** — surfaces block the grid behind them, creating depth without shadows.
